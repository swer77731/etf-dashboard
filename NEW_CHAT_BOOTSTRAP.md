# 給新對話 Claude 的啟動文

## 怎麼用

**開新的 Claude.ai 對話**(這樣 context 從乾淨開始),
複製下面整段貼進去當第一句話。

---

## 複製這段

```
我要開始做新專案:Taiwan ETF Dashboard 網站。

【背景】
我是個有 10+ 年經驗的台股交易者,今天從早到晚跟你的另一個對話一起做台指期策略,
做完一支真正達標的策略(Pullback_To_MA MC 635 萬),然後決定換方向做網頁。

【上一個對話學到的工作模式】
1. 我用 Claude Code 在自己電腦做事(不是只用對話)
2. CLAUDE.md 是跨 session 的紀律檔,放在專案資料夾
3. 我喜歡「下指令 → 離開電腦 → 回來看結果」的全自動模式
4. 我討厭 AI 一直問選擇題、討厭被打擾按 Y/N
5. 我直覺很準,常常抓出 AI 的邏輯偏差(像 long-only 鬼話、組合策略、多空共用參數等)
6. AI 寫的東西常常太「教科書」,我喜歡複雜、創新、有自己風格的解法

【新專案需求】
做一個 Taiwan ETF Dashboard 網站,含:
1. ETF 排行榜(從 FinMind 抓資料)
2. 自選日期區間多 ETF 績效比較圖表
3. 即時新聞牆
4. Telegram 推播服務(免費新聞 + 付費訊號分群)
5. 會員資料管理 + 訂閱金流

技術棧:FastAPI + SQLAlchemy + SQLite + APScheduler + HTMX + Alpine.js
       + Tailwind CDN + ECharts + python-telegram-bot
部署:Zeabur / Railway

設計:暗色系
- 背景 #0a0e1a / 卡片 #131829 / 邊框 #1f2937
- 文字主 #e5e7eb / 次 #9ca3af
- 漲紅 #ef4444 / 跌綠 #10b981 / 強調 #3b82f6

開發順序(不准跳):
骨架 → 排行榜 → 績效比較 → 新聞 → TG 推播 → 會員 → 金流

【你的角色】
- 幫我規劃路徑、判斷工具選擇
- 抓 Claude Code 在我電腦上做的偏差
- 不要叫我休息、不要問我「明天再說」
- 不要囉嗦,精準回答

【現在我要做的第一件事】
專案 CLAUDE.md 跟 Step 1 指令我已經有了(從前一個對話帶來),
我要把專案啟動。

問我:這個專案資料夾要建在哪個路徑?
```

---

## 為什麼要這樣開新對話

1. **這個對話 context 太滿**,AI 已經接近混淆狀態
2. **新對話從乾淨開始** = 反應快 + 不會搞錯主題
3. **這份啟動文濃縮了重點** = 新對話一開始就上路
4. 但**保留我的工作偏好**(不囉嗦、不問選擇題、要全自動)

---

## 操作順序

1. 開新對話 → 貼啟動文
2. 新對話會問你「資料夾建哪」
3. 你回答(例如 `C:\projects\etf_dashboard`)
4. 新對話會接著引導你 Step 1
5. 你照著做 Claude Code 操作
